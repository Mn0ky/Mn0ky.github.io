My Projects

OptiArchiver v2

Python3

A program that serves as a digital archiver of OptiFine versions. OptiArchiver allows for the preservation of this historic and wide-spanning mod. This includes auto-detection of new versions, auto-detection of version re-uploads, and the auto-downloading of said new versions directly to the file system with version names intact. OptiArchiver also includes discord webhook integration for instant notifications of any new downloads.

Personally, I keep the Python script running via a cronjob on my own Raspberry Pi. This cronjob is set to execute every single minute, 24/7!

Southern Lights

NodeJS, JavaScript and other technologies

This project integrates with a holiday light show. While the light show is running, it allows visitors to vote on a song that they would like to see and hear. The NodeJS code not only manages the voting but also communicates with the light show controller to monitor and control what is playing.

Stick Fight: QOL Mod

C#, .NET, and Unity

A quality-of-life mod which brings additions and improvements to Stick Fight: The Game. From a nice mod menu to chat commands, it even allows you to specify your own custom player color. There are many, many commands as well as a config allowing one to customize the game in even more ways.

Stick Fight: ExpandedWeapons

C#, .NET, and Unity

This mod for Stick Fight: The Game gives the player the ability to have the special/unused/dev weapons (such as the holy sword or mini minigun) to naturally spawn in along with the normal weapons. One can edit the rarities (spawn chances) of said weapons to decrease/increase the chance that they'll spawn. All changes to a weapon's active state and rarity are automatically saved to disk, which means changes are saved and loaded even after closing then reopening the game. It is important to note that this mod also adds or improves upon the functionality of these special weapons, such as with the bow.

Stick Fight: ThrowCalc

C#, .NET, and Unity

This mod for Stick Fight: The Game allows for rendering the path a weapon will take when thrown. The trajectory will begin being drawn as soon as the weapon is picked up. Once said weapon is thrown the trajectory will not be updated until a weapon is again picked up.
